# QR Studio

A professional, high-end, animated desktop application for **Windows** (and
cross-platform via Electron) that turns URLs into beautiful, fully-styled QR
codes. Built with Electron, React 18, TypeScript, Vite, Tailwind CSS and Framer
Motion, and packaged into a standalone `.exe` installer with electron-builder.

---

## Features

- Validated URL input with animated inline feedback (green check / red shake).
- Center-logo embedding via a native file dialog (PNG / JPG / SVG / WEBP), with
  automatic white rounded padding, a live size slider, and error-correction
  automatically forced to level **H (30%)** so the code stays scannable.
- Three color modes:
  - **Solid** - single foreground + background color, with preset swatches.
  - **Two-color** - primary color for data modules and secondary for the
    finder/eye patterns, or blended as a two-stop gradient.
  - **Rainbow** - a full-spectrum gradient with horizontal, vertical and radial
    directions.
- Module-shape options (square, rounded, dots, classy, extra-rounded) and
  custom eye styles.
- Large live preview that re-renders in real time with a smooth Framer Motion
  crossfade.
- High-resolution export to **PNG, SVG and JPG** at 512-2048 px, with an
  optional transparent background and an animated success toast.
- Frameless custom window with a draggable title bar (minimize / maximize /
  close), glassmorphism dark theme by default, and an animated light/dark
  theme toggle.
- Animated boot splash that fades into the UI, spring-physics micro-interactions
  throughout, and Inter bundled locally (no network fonts).

## Project structure

```
qr-generator-app/
  electron/
    main.ts          Electron main process + secure IPC handlers
    preload.ts       contextBridge bridge (window controls, dialogs, save)
  src/
    components/       UI components (title bar, cards, panels, preview, toast...)
    hooks/            useDebounce, useToast

    theme/           ThemeProvider (light/dark via CSS variables)
    utils/           URL validation/normalization
    App.tsx          Screen composition
    main.tsx         React entry (imports the bundled Inter font)
    index.css        Design system + Tailwind layers
  build/

    make_icon.py     Regenerates the icon
  index.html
  electron-builder.json
  vite.config.ts
  tailwind.config.js
  postcss.config.js
  tsconfig.json
  package.json
```

## Requirements

- **Node.js 18+** (Node 20 LTS recommended) and npm.
- **Windows** to build the Windows `.exe` targets (electron-builder builds
  Windows installers natively on Windows; on macOS/Linux you can build with
  Wine, but native Windows is simplest).
- Python 3 + Pillow only if you want to regenerate the icon (`build/icon.ico`
  is already included).

## Getting started (development)

```bash
# 1. Install dependencies
npm install

# 2. Launch the app in development (Vite + Electron with hot reload)
npm run dev
```

The first run may take a moment while Electron downloads its binary.

---

## Building the Windows .exe

```bash
# Type-check + build the renderer and Electron code, then package installers
npm run dist:win
```

Output is written to the `release/` folder:

- `QR-Studio-Setup-<version>.exe` - NSIS installer (lets the user choose the
  install location, creates Start-menu and desktop shortcuts).
- `QR-Studio-Portable-<version>.exe` - single-file portable executable.

To build for the current platform instead, run `npm run dist`.

## Available scripts

| Script            | What it does                                             |
| ----------------- | -------------------------------------------------------- |
| `npm run dev`     | Start Vite + Electron in development with hot reload.    |
| `npm run build`   | Type-check and build the renderer + main/preload.        |
| `npm run dist`    | Build and package installers for the current OS.         |
| `npm run dist:win`| Build and package the Windows NSIS + portable `.exe`.    |
| `npm run preview` | Preview the built renderer in the browser.               |
| `npm run typecheck`| Run the TypeScript compiler in no-emit mode.            |

## How it works

- **Security**: `contextIsolation` is on and `nodeIntegration` is off. The
  renderer only talks to the main process through the small, typed API exposed
  in `electron/preload.ts` (`window.api`). Native open/save dialogs and file I/O
  live entirely in the main process.
- **QR engine**: `src/qr/qrEngine.ts` is pure and converts the high-level
  `QrConfig` into `qr-code-styling` options. `useQrCode` keeps one canvas
  instance for the live preview and renders a fresh instance at the chosen size
  for export (SVG for vector output, canvas for PNG/JPG).
- **Theming**: colors are CSS variables defined in `index.css`; toggling the
  theme flips the `dark` class on `<html>`.
